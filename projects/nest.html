01 OVERVIEW

Tackling fast fashion waste through a circular economy.

The Problem: Fast fashion contributes significantly to environmental waste, yet many university students have closets full of well-preserved but unused clothes.

The Solution: NEST is a campus-based platform that integrates rental and purchase options. It allows students to monetize idle clothes while offering others a sustainable, low-cost way to experiment with new styles.

MY CONTRIBUTION

  • System Architecture: Designed the EERD and Relational Schema to support complex hybrid user roles (Buyer/Seller) and dual transaction modes (Rent/Buy).
  • Feature Implementation: Developed the "Seller Dashboard" for data visualization and the "Recommendation System" using collaborative filtering logic.
  • AI Integration: Built and integrated a Botpress chatbot to handle FAQs regarding sizing and rental policies, reducing user uncertainty.
02 SYSTEM DESIGN

Advanced Features

03 DATABASE ARCHITECTURE

Complex Transaction Logic

A core challenge was designing a schema that supports Hybrid Roles (a single user can be both a buyer and a seller) and Dual Transaction Modes (an item can be sold or rented).

  • Entity Integrity: Users are managed under a single `Member` entity but can trigger `Buyer` or `Seller` actions.
  • Transaction Logic: The `Transaction` entity handles both purchase and rental logic, linking to specific `Rental Records` for deadline tracking.

04 TECHNICAL CHALLENGES

Problem Solving

Challenge 1: Image Data Handling

Issue: We faced a discrepancy where images uploaded directly via SQL query displayed correctly, but images uploaded through the web interface failed to render properly.

Solution: We adjusted the strategy to store file paths in the database while keeping actual images in a server directory, improving performance and reliability.

Challenge 2: Chatbot Integration Costs

Issue: Initially planned to use ChatGPT API, but the cost was prohibitive.

Solution: Pivoted to Botpress. I designed conversation flows to query our internal FAQ database first, ensuring accurate answers without high API costs.
